---

## 1. 왜 필요한가

라벨에는 흔히 아래처럼 적힌다:

```
원재료: 정제수, 과일농축액(사과, 포도, 감초), 혼합제제, 구연산
```

이 때 `과일농축액`만 DB에 없고 하위 `사과/포도/감초` 중 **감초(restricted)** 또는 **별표3 금지 원료**가 섞여 있을 수 있음. 복합원재료는 반드시 **하위로 파고들어 재귀 매칭**해야 한다.

또, `restricted` 원료(별표2)는 DB의 `usage_condition`(조건 텍스트)을 **조건 유형별로 분류**하고 라벨·제조공정 정보로 **자동 검증**하거나 **담당자 확인 요청**으로 분기해야 한다.

---

## 2. 조건 유형 6가지 (newsamc + ambiguous)

| ConditionType | 정규식 패턴 | 자동 검증 가능? | 예시 |
|---------------|------------|---------------|------|
| `usage_purpose` | `용도\|목적\|사용\s*용도` | ❌ | "발효주 제조용에 한함" |
| `part_restriction` | `부위\|뿌리\|잎\|줄기\|종자\|열매` | ✅ `ingredient.part` 포함 여부 | "사용부위: 종실(볶은 것)" |
| `quantity_limit` | `함량\|%\|이하\|이상\|mg\|g\/kg\|ppm` | ⚠️ 기능1 Step 3(기준치) 영역 | "1일 섭취량 기준 관리" |
| `natural_synthetic` | `천연\|합성\|자연` | ❌ | "천연 원료에 한함" |
| `irradiation` | `조사\|방사선\|irradiat` | ❌ | "방사선 조사 비허용" |
| `ambiguous` | (기본값) | ❌ | 나머지 모두 |

> 우선순위는 **정의 순서**. 첫 번째로 매칭되는 패턴이 채택됨.

## 3. Python 구현 초안

```python
# backend/services/step1_ingredients_check.py (이어서)
import re
from typing import Literal, Optional
from pydantic import BaseModel

ConditionType = Literal[
    'usage_purpose', 'part_restriction', 'quantity_limit',
    'natural_synthetic', 'irradiation', 'ambiguous'
]

CONDITION_PATTERNS: list[tuple[re.Pattern, ConditionType]] = [
    (re.compile(r'용도|목적|사용\s*용도', re.I),             'usage_purpose'),
    (re.compile(r'부위|뿌리|잎|줄기|종자|열매', re.I),        'part_restriction'),
    (re.compile(r'함량|%|이하|이상|mg|g/kg|ppm', re.I),      'quantity_limit'),
    (re.compile(r'천연|합성|자연', re.I),                    'natural_synthetic'),
    (re.compile(r'조사|방사선|irradiat', re.I),              'irradiation'),
]


def classify_condition_type(conditions: str) -> ConditionType:
    if not conditions:
        return 'ambiguous'
    for pattern, ctype in CONDITION_PATTERNS:
        if pattern.search(conditions):
            return ctype
    return 'ambiguous'


class ConditionalEvaluation(BaseModel):
    ingredient_name: str
    condition_type: ConditionType
    condition_description: str
    is_satisfied: Optional[bool]    # None=판단불가 → 담당자 확인
    evidence: Optional[str] = None
```

## 4. 단건 평가 로직

```python
def evaluate_condition(
    ingredient_name: str,
    conditions: str,
    ingredient: Ingredient,
) -> ConditionalEvaluation:
    ctype = classify_condition_type(conditions)

    # 부위 제한: 자동 검증 가능
    if ctype == 'part_restriction' and ingredient.part:
        satisfied = ingredient.part.lower() in conditions.lower()
        return ConditionalEvaluation(
            ingredient_name=ingredient_name,
            condition_type=ctype,
            condition_description=conditions,
            is_satisfied=satisfied,
            evidence=f"신고 부위: {ingredient.part}",
        )

    # 함량 제한: 기능1 Step3(기준치) 영역이므로 evidence만 기록
    if ctype == 'quantity_limit' and ingredient.percentage is not None:
        return ConditionalEvaluation(
            ingredient_name=ingredient_name,
            condition_type=ctype,
            condition_description=conditions,
            is_satisfied=None,    # Step3에서 최종 판정
            evidence=f"신고 함량: {ingredient.percentage}%",
        )

    # 나머지: 자동 판단 불가
    return ConditionalEvaluation(
        ingredient_name=ingredient_name,
        condition_type=ctype,
        condition_description=conditions,
        is_satisfied=None,
    )
```

## 5. 일괄 평가

`match-chain`에서 나온 `restricted` 결과들을 일괄로 평가:

```python
def evaluate_conditional_ingredients(
    match_results: list[IngredientMatchResult],
    ingredients: list[Ingredient],
) -> list[ConditionalEvaluation]:
    evals: list[ConditionalEvaluation] = []
    for r in match_results:
        if r.verdict == 'restricted' and r.conditions:
            ing = next((i for i in ingredients if i.name == r.ingredient.name),
                       r.ingredient)
            evals.append(evaluate_condition(
                r.ingredient.name, r.conditions, ing))
    return evals
```

## 6. 복합원재료 재귀 검증

```python
async def evaluate_compound_ingredients(
    db: Connection, ingredients: list[Ingredient],
) -> tuple[list[IngredientMatchResult], list[ConditionalEvaluation], list[dict]]:
    all_sub_results: list[IngredientMatchResult] = []
    conditional_evals: list[ConditionalEvaluation] = []
    escalations: list[dict] = []

    for ing in ingredients:
        if ing.sub_ingredients:
            sub_agg = await run_ingredient_match_chain(db, ing.sub_ingredients)
            all_sub_results.extend(sub_agg.results)
            escalations.extend(sub_agg.escalations)

            if sub_agg.prohibited > 0:
                escalations.append({
                    "module_id": "F1",
                    "trigger_type": "compound_prohibited",
                    "reason": (f'복합원재료 "{ing.name}"의 하위 성분 중 '
                               f'금지 원료 {sub_agg.prohibited}건 감지'),
                })
    return all_sub_results, conditional_evals, escalations
```

SYNTHETIC_FLAVOR_KEYWORDS = ["합성향료", "synthetic flavor", "artificial flavoring"]

def is_synthetic_flavor(name: str) -> bool:
    lowered = name.lower()
    return any(kw.lower() in lowered for kw in SYNTHETIC_FLAVOR_KEYWORDS)


# run_feature1 안에서 Step 1 직전에 분기
for ing in ingredients:
    if is_synthetic_flavor(ing.name):
        results.append(IngredientMatchResult(

## 8. 최종 통합 흐름

```
[ingredients]
    │
    ▼
[Step 0] forbidden_ingredients 게이트  ← 02 문서
    │
    ▼
[Step 1] 5단계 매칭 (run_ingredient_match_chain)  ← 02 문서
    │
    ▼
[Step 1-A] 복합원재료 재귀 (evaluate_compound_ingredients)
    │
    ▼
[Step 1-B] 조건부(restricted) 평가 (evaluate_conditional_ingredients)
    │
    ▼
[Step 2] 기준치 수치 비교 → 04 문서
```

## 9. 테스트 시나리오

| 케이스 | 입력 | 기대 |
|-------|------|-----|
| 복합 정상 | `과일농축액(사과,포도)` | 하위 2건 모두 permitted |
| 복합 금지 | `한약 엑기스(초오)` | 하위 초오 prohibited → 에스컬레이션 |
| 부위 제한 적합 | `{name:"은행", part:"종실"}` → 조건 "사용부위: 종실(볶은 것)" | `is_satisfied=True` |
| 부위 제한 불합 | `{name:"은행", part:"잎"}` → 조건 "사용부위: 종실" | `is_satisfied=False` |
| 함량 제한 | `{name:"감초", percentage:2}` → 조건 "글리시리진산 함량 관리" | `is_satisfied=None` (Step3로 위임) |
| 애매 | 조건 "전문가 자문 후 사용" | `ambiguous`, `is_satisfied=None` |
| 합성향료 | `{name:"합성향료(딸기향)"}` | 담당자에게 하위원료 요청 |
