## 2. 공통 설계 원칙 (팀컨벤션 §6·§7 준수)

- **API 호출은 반드시 `lib/api.ts` 경유** — 컴포넌트에서 직접 `fetch` 금지
- `any` 금지 → `unknown` 또는 구체 타입
- 컴포넌트명 PascalCase, 함수명 camelCase
- `interface` 우선 사용
- 에러 응답 처리: `{error, message, feature}` 형식 파싱 후 토스트 표시
- 한국어 에러 메시지는 그대로 표시, 영어 에러 코드는 내부 로깅용

## 8. P5 — `admin/law-update/page.tsx`

### 플로우

```
[1. 파일 선택 (HWP/PDF/XLSX)]
     ↓
[2. 법령명 선택 (드롭다운: 식품첨가물공전, 식품공전 등)]
     ↓
[3. 업로드 버튼] → POST /admin/law/upload → job_id 반환
     ↓
[4. 추출 진행 상태] (폴링 or SSE)
     ↓
[5. 추출 결과 테이블] — extracted + needs_review
  - 각 행 편집 가능
  - 체크박스로 thresholds 반영 여부 선택
     ↓
[6. 검수 완료 버튼] → POST /admin/law/extract-confirm
     ↓
[7. 결과 스낵바] "N건 thresholds 반영 완료"
```
