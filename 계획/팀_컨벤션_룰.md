## 3. 브랜치 전략

```
main              ← 배포용. 직접 push 금지. PR로만 병합.
feature/이름-설명  ← 개발 브랜치. 완료되면 main으로 PR.
```

### 브랜치명 규칙

```
feature/{이름}-{기능설명-하이픈연결}

예시:
feature/병찬-import-check
feature/병찬-law-extractor
feature/아람-food-type

feature/공통-parser-service
feature/공통-pipeline
```

- `feature → main`: PR 생성, 해당 담당자 + 1명 approve 후 병합

## 4. 커밋 메시지 규칙

```
{type}: {한국어 설명}

type:
  feat     새 기능
  fix      버그 수정
  db       DB 스키마·초기 데이터 변경
  prompt   Claude 프롬프트 수정
  refactor 기능 변화 없는 코드 정리
  chore    설정·패키지·환경변수
  docs     문서
```

### 예시

```
feat: 포지티브 리스트 학명·INS 번호 매칭 추가
feat: 식품유형 분류 주류 분기 로직 구현
feat: 수입 필요서류 OEM·친환경 조건 분기 처리
prompt: 수출국 라벨 부적절 문구 탐지 프롬프트 개선
feat: 14대 알레르기 자동 감지 로직
db: thresholds 음료류 보존료 기준치 50개 입력
chore: requirements.txt rapidfuzz 추가
```

### 금지

- `fix: 수정`, `update`, `asdf` 등 내용 없는 커밋
- 서로 다른 기능을 한 커밋에 묶기

## 8. DB 변경 규칙

### schema.sql 변경 시

1. 채팅에 변경 내용 공지 (어떤 테이블, 어떤 컬럼)
2. `schema.sql` 수정 + PR 오픈, **전원 Reviewer 지정**
3. 전원 approve 후 Supabase 대시보드에서 실행 → 완료 공지

### 컬럼 변경 원칙

| 작업 | 허용 여부 |
|------|----------|
| 새 컬럼 추가 (DEFAULT 포함) | ✅ |
| 새 컬럼 추가 (NOT NULL, DEFAULT 없음) | ❌ |
| 기존 컬럼 타입 변경 | ❌ |
| 기존 컬럼 삭제 | ❌ (전원 합의 후 예외) |

### 담당자별 DB 초기 데이터 입력 책임

| 담당자 | 입력 테이블 | 출처 |
|--------|------------|------|
| **병찬** | `thresholds`, `ingredient_list` | 4가지 공전, 식품원료목록, 한시적 기준 및 규격 인정 기준 |
| **아람** | `required_documents` (식품유형 매핑) | 식품공전 식품유형 분류원칙, 주세법 시행령 별표1·3 |
| **기능3 담당** | `required_documents` (서류 목록) | 수입신고 구비서류 목록, OEM수입식품관리, 동등성인정 |
| **세연** | `allergy_list`, `additive_label_rules`, `gmo_ingredients`, `label_rules` | 식품등의 표시기준, 식품 등의 표시·광고에 관한 법률 시행규칙 별표 |

> 데이터 입력 시: `law_source` 컬럼 필수 기재, `is_verified=true` 로 입력 후 팀 채팅에 완료 공지.

## 11. 기능 간 의존 관계

아래 순서를 지키지 않으면 다음 기능 테스트가 불가능하다.

```
[기능 1] 수입 가능 여부 판정  (병찬)
     ↓  결과: 원재료 목록 + 수입가능 여부
[기능 2] 식품유형 분류        (아람)
     ↓  결과: 식품유형명
[기능 3] 수입 필요서류 안내   (미정)   ← 기능 2 결과 필요
[기능 4] 수출국표시사항 검토  (미정)   ← 기능 1 원재료 목록 필요
     ↓
[기능 5] 한글표시사항 검토    (세연)   ← 기능 1·2 결과 모두 필요
```

### 초반 블로킹 주의

| 완료 기한 | 작업 | 안 되면 |
|----------|------|---------|
| Day 1 | Supabase schema.sql 실행, 환경변수 전원 공유 | 전체 개발 불가 |
| Day 1 | parser-service 로컬 실행 확인 | 서류 파싱 불가 |
| Day 2 | `ingredient_list`, `thresholds` 기본 데이터 입력 (병찬) | 기능 1 테스트 불가 |
| Day 2 | `required_documents` 기본 데이터 입력 | 기능 3 테스트 불가 |
| Day 3 | SSE 파이프라인 틀 완성 (공통 인프라) | 기능 연결 불가 |
