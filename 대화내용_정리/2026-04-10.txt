▣ 발견된 갭 (7가지)

[갭 1] 수입 필요서류 안내 기능 완전 누락 🔴
- 회사_요구사항.md 항목 2번에 명시된 기능
- 구비서류 목록, OEM수입식품, 친환경 동등성 인정 서류 안내

[갭 2] 수출국표시사항 검토 기능 누락 🔴
- 회사_요구사항.md 항목 3번 + 설문 1차 C1 1순위 기능
- 해외 원본 라벨 이미지 업로드 → 부적절 문구·그림 검토
- 서류-라벨 내용 교차 확인 (제품명, 원재료, 내용량, 원산지, 제조사)

[갭 3] STEP3 복합 합산 판정 누락 🟠
- 설문 2차 F2: 착색료 2종 이상 사용 시 합산 기준치도 확인 필요
- 현재 계획은 개별 성분 1:1 비교만 있음

[갭 4] INS/CAS 번호 기반 성분 매칭 없음 🟠
- 설문 2차 D3: 라벨 성분명 ≠ 법령 등재명일 때 INS/CAS로 매칭
- ingredient_list, material_codes 테이블에 관련 컬럼 없음

[갭 5] 재수입 케이스 재사용 기능 없음 🟡
- 설문 2차 H2: 재수입 시 이전 결과 참고 후 변경사항만 확인
- cases 테이블에 parent_case_id 없음

[갭 6] 다국어 번역 구현 명세 없음 🟡
- 설문 1차 B9: 영어 70%, 중·일·유럽어 25%
- DEEPL_API_KEY만 있고 실제 구현 명세 없음

[갭 7] 합성향료 하위원료 확인 로직 없음 🟡
- 설문 2차 F1: 합성향료는 하위원료까지 식품첨가물공전 [별표1] 확인 필요

[주제 2] INS/CAS 번호 — DB_최신 폴더 검토 결과
----------------------------------------------------------------

▣ 결론: DB_최신 폴더에 INS/CAS 정리된 파일 없음
- 식품첨가물 품목별 리스트 PDF 5개 확인 → 한국어명 + 영문명 + 내부 번호만 있음
- INS/CAS 번호 없음

▣ 단, 사용자가 확인한 사항:
- DB_최신/4_고시(공전)/(식품첨가물공전)/IV.품목별 성분규격.hwpx 파일에
  INS 번호 + CAS 번호가 포함됨 (사진으로 확인)
  예: 가교카복시메틸셀룰로스나트륨 → INS: 468, CAS No.: 74811-65-7

▣ 활용 방안:
- kordoc으로 HWPX 파싱 → 정규식(INS:\s*(\d+), CAS No\.\s*([\d-]+))으로 추출
- ingredient_list 테이블의 ins_number, cas_number 컬럼에 저장
- 보조 소스: Codex Alimentarius CXG 36 (FAO 사이트 무료 다운로드)

[주제 3] 파이프라인 순서 논의 — 원재료 허용 여부 vs 식품유형 분류
----------------------------------------------------------------

▣ 결론: 원재료 허용 여부 판정이 먼저 맞음

▣ 근거:
1. 회사_요구사항.md: "수입가능 여부 판정" → "식품유형 분류" 순서 명시
2. 직원_설문_2차.pdf A1: 실제 업무 순서도 원재료 허용 여부 1순위
3. 논리: 포지티브 리스트는 식품유형 무관한 글로벌 허용 목록
   → 수입 불가 원료 있으면 즉시 중단, 이후 단계 낭비 없음
4. 기준규격(STEP3)만 식품유형에 의존 → STEP2(식품유형) 이후에 배치하면 됨

▣ 변경된 파이프라인 순서:
  기존: STEP1(식품유형) → STEP2(원재료) → STEP3(기준규격) → ...
  변경: STEP1(원재료허용) → STEP2(식품유형) → STEP_A(수입필요서류)
        → STEP3(기준규격) → STEP4(공정코드) → STEP5(성분코드)
        → STEP_B(수출국라벨검토) → STEP6(한글라벨)
