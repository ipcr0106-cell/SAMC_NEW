# f0 (경아) 수정 요청 사항

> F1 병합 과정에서 발견된 f0 OCR/파싱 보완 요청입니다.
> 작성일: 2026-04-16

---

## 1. 원재료 사용 부위 (`part`) 추출 추가

**현재**: `IngredientItem`에 `part` 필드 없음
**필요**: F1이 조건부 원료(restricted) 판정 시 "잎만 허용", "뿌리 사용 금지" 같은 부위 조건을 평가함

**수정 위치:**
- `backend/schemas/upload.py` — `IngredientItem`에 `part: str = ""` 필드 추가
- `backend/services/parsing_service.py` — Claude 프롬프트에 부위 추출 지시 추가

**예시:**
```
원재료배합비율표에 "스테비아(잎)" 이라고 되어 있으면
→ name: "스테비아", part: "잎"
```

**긴급도**: 중간 — 없으면 F1이 부위 조건을 자동 판정 못하고 "담당자 확인 필요"로 처리

---

## 2. 복합원재료 하위 성분 (`sub_ingredients`) 파싱

**현재**: 원재료를 flat 리스트로만 추출
**필요**: F1 Step 1-A에서 복합원재료를 재귀적으로 검증함 (예: "과일혼합(딸기60%, 블루베리40%)")

**수정 위치:**
- `backend/schemas/upload.py` — `IngredientItem`에 `sub_ingredients: list[IngredientItem] = []` 추가
- `backend/services/parsing_service.py` — 프롬프트에 중첩 구조 파싱 지시 추가

**예시:**
```json
{
  "name": "과일혼합",
  "ratio": "30",
  "sub_ingredients": [
    { "name": "딸기", "ratio": "60" },
    { "name": "블루베리", "ratio": "40" }
  ]
}
```

**긴급도**: 높음 — 복합원재료가 포함된 제품에서 금지 원료가 하위에 숨어있으면 F1이 탐지 못함

---

## 3. 알코올 도수 (`alcohol_percentage`) 추출

**현재**: ParsedResult에 알코올 도수 필드 없음
**필요**: F1 Step 3에서 주류 여부 판정 및 주류 전용 기준치 적용에 사용

**수정 위치:**
- `backend/schemas/upload.py` — `BasicInfo` 또는 `LabelInfo`에 `alcohol_percentage: Optional[float] = None` 추가
- `backend/services/parsing_service.py` — 프롬프트에 라벨/서류에서 알코올 도수 추출 지시

**예시:**
```
라벨에 "ALC. 14.5% BY VOL" → alcohol_percentage: 14.5
```

**긴급도**: 높음 — 주류 제품의 경우 기준치가 완전히 달라지므로 도수 없이는 정확한 판정 불가

---

## 4. 내용량 (`content_volume`) 추출 — F4에서도 요청

**현재**: ParsedResult에 내용량 필드 없음
**필요**: F4 교차검증에서도 필요, F1에서는 직접 사용하지 않으나 향후 필요 가능

**수정 위치:**
- `backend/schemas/upload.py` — `BasicInfo` 또는 `LabelInfo`에 `content_volume: str = ""` 추가

**긴급도**: 낮음 (F1 기준) / 중간 (F4 기준)

---

## 5. 제조사 (`manufacturer`) 추출 — F4에서도 요청

**현재**: ParsedResult에 제조사 필드 없음
**필요**: F4 교차검증

**수정 위치:**
- `backend/schemas/upload.py` — `BasicInfo`에 `manufacturer: str = ""` 추가

**긴급도**: 낮음 (F1 미사용) / 중간 (F4 기준)
