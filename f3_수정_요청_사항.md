# F3 (유빈) 수정 요청 사항

> F3 병합 과정에서 발견된 확인/보완 요청입니다.
> 작성일: 2026-04-16

---

## 1. Supabase 테이블 확인 (필수)

F3가 사용하는 테이블이 Supabase에 존재하는지 확인해주세요:

- `f3_required_documents` (52건) — 서류 매칭 규칙
- `f3_country_groups` (181건) — 국가 그룹 매핑 (BSE_36, ASF_73 등)
- `f3_keyword_synonyms` (38건) — OCR/영문 원재료명 → DB 정규화 매핑

우리 환경에서 아직 생성되지 않았다면 시드 스크립트 실행 필요.

---

## 2. Pinecone 인덱스 확인 (필수)

- 인덱스명: `samc-law-f3` (85개 청크)
- 임베딩: multilingual-e5-large (Pinecone inference API 사용)
- 시딩 스크립트: `feature_3/frontend/scripts/seed-pinecone.ts` (TypeScript)

우리 환경에서 Pinecone에 데이터가 올라가 있는지 확인 필요.

## 4. category (7대 구분) 매핑 검토

F3의 `ProductInfo.category`는 식약처 7대 구분(축산물/수산물/가공식품/농.임산물/식품첨가물/기구또는용기.포장/건강기능식품)입니다. 현재 F2 출력에는 이 값이 없고 `category_name`(대분류)만 있습니다.

F3 코드는 `category` 없이도 `is_livestock_food_type()` 등으로 축산물 여부를 판별하므로 핵심 로직은 돌아갑니다. 다만 정확도를 높이려면:
- F2의 `category_name`을 7대 구분으로 매핑하는 로직 추가 검토
- 또는 `category` 필드를 Optional로 유지하고 없으면 food_type 기반 추론

## 5. 프론트엔드 API 경로 수정 필요

`RequiredDocsPage.tsx`가 호출하는 API 경로가 우리 파이프라인과 다릅니다.

**현재 (유빈 원본):**
- 프론트에서 직접 `ProductInfo` JSON을 구성해서 `/api/v1/required-docs`에 POST
- 별도의 `/api/query-docs`, `/api/parse-document` 등 Next.js API Route 사용

**변경 필요 (우리 파이프라인):**
- 프론트는 `POST /cases/${caseId}/pipeline/feature/3/run`만 호출
- 백엔드가 f0/F1/F2 결과를 자동으로 조회해서 F3 실행

## 6. 미복사 API Route — 필요 시 복사 가능

아래 API Route는 F3에 있었지만 현재 복사하지 않았습니다:

| Route | 역할 | 복사 여부 | 이유 |
|-------|------|:---------:|------|
| `/api/ai-cross-check` | AI 독립 판정 (교차검증) | **복사됨** | 가치 있는 기능 |
| `/api/explain-docs` | 서류 필요 이유 설명 생성 | **복사됨** | 가치 있는 기능 |
| `/api/query-docs` | Python 백엔드 프록시 | 미복사 | 우리 feature3.py가 대체 |
| `/api/parse-document` | 서류 OCR 파싱 | 미복사 | f0 upload.py가 대체 |
| `/api/analyze-ingredients` | 원재료 동의어 매핑 | 미복사 | 필요하면 추가 가능 |

`/api/analyze-ingredients`는 원재료명을 DB 키워드로 매핑하는 기능입니다. F3에서 키워드 매칭 정확도를 높이고 싶다면 복사 요청하세요.

## 7. f0 + F1 + F2 → F3 데이터 파이프라인 — PM이 임의로 구현한 부분

> **주의**: 이 연결은 PM이 병합 과정에서 임의로 작성한 것입니다.

### 현재 구현된 입력값 매핑

```
f0 (step_key='0')
├── basic_info.export_country    → origin_country (필수)
├── basic_info.is_oem            → is_oem
├── basic_info.is_first_import   → is_first_import
└── basic_info.is_organic        → has_organic_cert

F1 (step_key='1')
└── ingredients[].name           → product_keywords

F2 (step_key='2')
├── food_type                    → food_type (필수)
├── category_name                → food_large_category
└── subcategory_name             → food_mid_category
```

### F3 출력값 (pipeline_steps step_key='A' → ai_result)

```
├── food_type           "위스키"
├── origin_country      "영국"
├── is_first_import     false
├── submit_docs[]       제출 서류 목록
│   ├── doc_name        "수입신고서"
│   ├── doc_description "수입식품 신고 시 작성하는 기본 서류"
│   ├── is_mandatory    true
│   ├── submission_type "submit" | "keep"
│   ├── submission_timing "every" | "first"
│   ├── law_source      "수입식품안전관리 특별법 시행규칙 제27조"
│   ├── match_reason    "공통 서류 (모든 수입식품)"
│   └── decision_axis   "공통" | "식품유형" | "국가" | ...
├── keep_docs[]         보관 서류 목록 (같은 구조)
├── total_submit        제출 서류 수
├── total_keep          보관 서류 수
├── warnings[]          경고 메시지
└── match_confidence    "high" | "needs_review"
```

### 수정하고 싶다면

`backend/routers/feature3.py`에서 PM이 추가한 함수:

```
def _fetch_pipeline(case_id, step_key) → dict
  └─ pipeline_steps 조회 (단순 DB 조회)

def _build_product_info_from_pipeline(case_id) → ProductInfo
  └─ ⭐ 입력 매핑을 바꾸고 싶으면 이 함수를 수정
```

**완전히 걷어내고 싶다면:**
1. 위 2개 함수 + `/cases/{case_id}/pipeline/feature/3/run` 엔드포인트 삭제
2. 원본 `/api/v1/required-docs` 엔드포인트를 다시 추가
3. 프론트에서 직접 ProductInfo를 구성해서 호출

## 8. F2의 f2_required_docs.py (stub) 삭제됨

F2 병합 시 만들었던 간이 필요서류 조회(`f2_required_docs.py`)는 F3가 본격 구현이므로 **삭제되었습니다.** main.py에서도 제거됨. F3의 `feature3.py`가 대체합니다.

## [공통] 법령 DB 전처리 — HWPX/PDF 파일 처리 확인

법령 원본 파일이 HWPX(한글) 또는 PDF 형식으로 제공됩니다. 자기 기능의 법령 DB 전처리 스크립트가 **두 형식 모두 처리 가능한지** 확인해주세요.

**확인 사항:**
- PDF 파싱: `pdfplumber`, `PyMuPDF(fitz)`, `pdfminer` 등 사용 가능한지
- HWPX 파싱: `python-hwp`, 외부 파서 서비스, 또는 직접 XML 파싱 가능한지
- 전처리 스크립트에서 파일 확장자에 따라 분기 처리가 되어 있는지
- 법령 업데이트 시스템(`/admin/law-update`)에서 PDF/HWPX 업로드 시 자기 기능의 전처리가 정상 트리거되는지

**현재 상태:**
- f0: PDF(PyMuPDF) + HWPX(외부 파서 서비스) 지원
- F4: PDF(PyMuPDF) + HWPX 지원 (`preprocess_laws.py`)
- F1/F2/F3/F5: 각 담당자가 확인 필요
