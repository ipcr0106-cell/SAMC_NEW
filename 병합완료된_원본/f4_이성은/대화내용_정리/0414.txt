── 3. 법령 개정 자동 업데이트 시스템 ──────────────────────────────────────

[신규 파일] backend/db/feature4/extract_image_violation_types.py
  - 관리자가 법령 PDF를 업로드하면 자동 실행
  - 1단계: Claude가 법령 조문에서 이미지 위반 후보 추출
  - 2단계: 3가지 기준으로 자동 검증
      ① 이미지 전용성 (텍스트가 아닌 이미지·그림·마크·심볼 위반)
      ② 법령 근거 명확성 (실제 한국 법령 조문 근거)
      ③ 기존 유형 차별성 (이미 등록된 유형과 다른 새 위반)
      ※ ④ AI 탐지 가능성은 기준에서 제외 (법령상 유효한 유형 모두 포함, AI가 불확실하면 사용자에게 직접 확인 요청)
  - confidence >= 0.8 + 3가지 통과 → is_active=True 자동 활성화
  - 미통과 → is_active=False + 사유 기록 (pending)
  - 기존 유형 보완 항목 → 검증 없이 sub_items에 병합

[신규 DB 테이블] f4_image_violation_types
  - type_name, sub_items, default_severity, severity_condition, law_ref
  - source: 'seed' | 'auto' | 'manual'
  - is_active: boolean
  - review_note: 자동 활성화/보류 사유

[신규 파일] backend/db/feature4/seed_image_violation_types.py
  - 위 26가지 초기 유형을 DB에 삽입 (source='seed', is_active=True)
  - 중복 type_name은 스킵
  - 실행: python seed_image_violation_types.py


── 4. 동적 프롬프트 생성 (backend/routers/feature4.py) ──────────────────────

  - _build_image_prompt(supabase): DB에서 is_active/is_pending 유형 읽어 프롬프트 동적 생성
  - 활성 유형 → "확정 위반 유형" 섹션 → review_level="confirmed"
  - 보류 유형 → "검토 권고 유형" 섹션 → review_level="suggested"
    (AI가 불확실하면 reasoning에 "AI 판단 불확실 — 직접 확인 권고" 명시)
  - _prompt_cache: 모듈 레벨 캐시, 법령 업로드 시 _invalidate_prompt_cache()로 무효화


── 5. 관리자 API (backend/routers/admin_laws.py) ──────────────────────────

  GET  /admin/laws                              — 법령 목록 조회
  POST /admin/laws/upload                       — 법령 PDF 업로드 + 자동 처리
    → extract_image_types_for_law() 자동 실행
    → _invalidate_prompt_cache() 자동 호출
    → 응답에 image_types_pending, image_types_supplemented 포함
  GET  /admin/laws/image-violation-types        — 위반 유형 전체/활성 목록 조회
    → ?active_only=true 파라미터로 활성만 필터
  PATCH /admin/laws/image-violation-types/{id}/activate — 보류 유형 수동 활성화
