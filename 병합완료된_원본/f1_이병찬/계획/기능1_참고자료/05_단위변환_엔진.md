## 1. 두 가지 역할

SAMC_NEW의 단위 변환은 **두 레이어**가 있다:

| 레이어 | 대상 | 예시 |
|------|------|------|
| **A. 텍스트 레벨** (라벨 OCR 결과 정규화) | `16 oz` → `453.59g (16 oz)` | 기능4의 입력 전처리 |
| **B. 수치 레벨** (기준치 비교) | 사용자 입력 `0.05%` → 기준치 단위 `g/kg` 환산 | 기능1 Step 3 |

newsamc는 A만 구현. SAMC_NEW는 **B 구현이 더 중요** (기능1 핵심).

## 2. 텍스트 레벨 변환 (newsamc 이식)

### 변환 규칙표

| 단위 | 변환 | 상수 | 정규식 |
|-----|------|------|--------|
| oz (무게) | × 28.3495 = g | `OZ_TO_G` | `/(\d+(?:\.\d+)?)\s*oz\b/gi` |
| lb, lbs (무게) | × 453.592 = g | `LB_TO_G` | `/(\d+(?:\.\d+)?)\s*lb[s]?\b/gi` |
| fl oz (부피) | × 29.5735 = ml | `FL_OZ_TO_ML` | `/(\d+(?:\.\d+)?)\s*fl\.?\s*oz\b/gi` |
| gal, gallon (부피) | × 3785.41 = ml | `GAL_TO_ML` | `/(\d+(?:\.\d+)?)\s*gal(?:lon)?[s]?\b/gi` |
| °F (온도) | (v-32) × 5/9 = °C | — | `/(\d+(?:\.\d+)?)\s*°F\b/gi` |

### Python 구현

```python
# backend/utils/unit_converter.py
import re
from typing import Literal
from pydantic import BaseModel

ConversionType = Literal['weight', 'volume', 'temperature']

OZ_TO_G     = 28.3495
LB_TO_G     = 453.592
FL_OZ_TO_ML = 29.5735
GAL_TO_ML   = 3785.41
DECIMAL_PLACES = 2


def _r2(v: float) -> float:
    return round(v, DECIMAL_PLACES)


class UnitConversion(BaseModel):
    original: str
    converted: str
    type: ConversionType


_CONVERSION_RULES: list[dict] = [
    {"pattern": re.compile(r'(\d+(?:\.\d+)?)\s*oz\b', re.I),
     "type": "weight",
     "convert": lambda v: (f"{_r2(v * OZ_TO_G)}g", "g")},
    {"pattern": re.compile(r'(\d+(?:\.\d+)?)\s*lb[s]?\b', re.I),
     "type": "weight",
     "convert": lambda v: (f"{_r2(v * LB_TO_G)}g", "g")},
    {"pattern": re.compile(r'(\d+(?:\.\d+)?)\s*fl\.?\s*oz\b', re.I),
     "type": "volume",
     "convert": lambda v: (f"{_r2(v * FL_OZ_TO_ML)}ml", "ml")},
    {"pattern": re.compile(r'(\d+(?:\.\d+)?)\s*gal(?:lon)?[s]?\b', re.I),
     "type": "volume",
     "convert": lambda v: (f"{_r2(v * GAL_TO_ML)}ml", "ml")},
    {"pattern": re.compile(r'(\d+(?:\.\d+)?)\s*°F\b', re.I),
     "type": "temperature",
     "convert": lambda v: (f"{_r2((v - 32) * 5 / 9)}°C", "°C")},
]


def convert_units_in_text(text: str) -> tuple[str, list[UnitConversion]]:
    """텍스트 내 영미 단위를 한국 표준 단위로 변환. (변환후 텍스트, 기록 배열) 반환."""
    conversions: list[UnitConversion] = []
    converted = text

    for rule in _CONVERSION_RULES:
        def _sub(m: re.Match) -> str:
            try:
                num = float(m.group(1))
            except ValueError:
                return m.group(0)
            replacement, _ = rule["convert"](num)
            conversions.append(UnitConversion(
                original=m.group(0).strip(),
                converted=replacement,
                type=rule["type"],
            ))
            return f"{replacement} ({m.group(0).strip()})"
        converted = rule["pattern"].sub(_sub, converted)

    return converted, conversions
```

## 3. 수치 레벨 변환 (기능1 핵심)

### 기본 단위 변환표

| 원본 | 대상 | 환산 |
|-----|------|-----|
| `%` | `g/kg` | `pct * 10` |
| `%` | `mg/kg` | `pct * 10000` |
| `g/kg` | `mg/kg` | `val * 1000` |
| `mg/kg` | `g/kg` | `val / 1000` |
| `g/kg` | `%` | `val / 10` |
| `mg/L` | `mg/kg` | (밀도 1.0 가정, 정확한 환산 필요 시 제품별 밀도) |
| `ppm` | `mg/kg` | `val` (동일) |

### 환산계수 (conversion_factor) 개념

일부 성분은 **염 형태 → 산 형태**로 환산이 필요하다. 예:

- 안식향산나트륨 → 안식향산 기준치 비교 시 × **0.847**
- 소르빈산칼륨 → 소르빈산 기준치 비교 시 × **0.746**

**저장 위치**: `thresholds.conversion_factor NUMERIC(8,6)` 컬럼 (§01 옵션 A).

### Python 구현

```python
def convert_unit(
    value: float,
    source_unit: str,
    target_unit: str,
    factor: Optional[float] = None,   # 염→산 환산계수 (DB의 conversion_factor)
) -> float:
    """
    수치 단위 변환. factor가 있으면 먼저 적용 (염→산 환산).
    """
    v = value * (factor if factor else 1.0)

    # 동일 단위
    if source_unit == target_unit:
        return v

    # 정규화 키
    s = source_unit.lower().strip()
    t = target_unit.lower().strip()

    # % ↔ g/kg ↔ mg/kg
    if s == "%":
        base_g_per_kg = v * 10.0
        if t == "g/kg":  return base_g_per_kg
        if t == "mg/kg": return base_g_per_kg * 1000.0
    if s == "g/kg":
        if t == "mg/kg": return v * 1000.0
        if t == "%":     return v / 10.0
    if s == "mg/kg" or s == "ppm":
        if t == "g/kg":  return v / 1000.0
        if t == "%":     return v / 10000.0

    raise ValueError(f"Unsupported conversion: {source_unit} → {target_unit}")
```

## 4. 기준치 측의 환산계수 저장 전략

개발계획서 v6는 `condition_text` TEXT 필드에 환산 정보를 녹여 넣자고 했지만, 파싱 리스크 있음. **newsamc 설계대로 별도 숫자 컬럼 권장**.

```sql
-- §01 제안대로 thresholds 보강
ALTER TABLE thresholds
  ADD COLUMN IF NOT EXISTS conversion_factor NUMERIC(8,6);

-- 예: 안식향산나트륨의 안식향산 환산
INSERT INTO thresholds
  (ingredient_name, food_type, threshold_value, unit,
   condition_text, conversion_factor, is_verified, law_source)
VALUES
  ('안식향산나트륨', '과채음료', 0.6, 'g/kg',
   '안식향산 기준. 안식향산나트륨 측정시 ×0.847',
   0.847, true, '식품첨가물공전');
```

## 5. 단위 테스트 시나리오

| 케이스 | 입력 | 기대 |
|-------|------|------|
| oz → g | `"16 oz chicken"` | `"453.59g (16 oz) chicken"` |
| °F → °C | `"Bake at 350°F"` | `"Bake at 176.67°C (350°F)"` |
| % → g/kg | `convert_unit(0.05, "%", "g/kg")` | `0.5` |
| 환산계수 | `convert_unit(0.5, "g/kg", "g/kg", factor=0.847)` | `0.4235` |
| % → mg/kg | `convert_unit(0.01, "%", "mg/kg")` | `100.0` |
| 미지원 | `convert_unit(1, "cup", "ml")` | ValueError |
