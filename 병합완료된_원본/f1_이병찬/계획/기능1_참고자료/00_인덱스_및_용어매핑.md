## ⚠️ 중요 원칙

1. **newsamc 코드를 그대로 복사하지 않는다.** newsamc는 TypeScript + Next.js API Route, SAMC_NEW는 FastAPI + Python. 로직·스키마·시드만 추출해서 Python으로 재작성.
2. **개발계획서 네이밍을 우선한다.** 충돌 시 개발계획서 v6의 파일명·테이블명·컬럼명이 정식 표기.
3. **공유 파일 수정은 금지.** `schema.sql`, `main.py`, `supabase_client.py`, `lib/api.ts` 등은 팀_컨벤션_룰 §2-4에 따라 전원 합의 필요. 이 MD들은 **초안 제안**일 뿐, 단독 커밋 금지.
4. **검수(`is_verified=true`) 강제.** 개발계획서 §8, 리스크 3에 따라 모든 임계값·기준치는 `is_verified=true` 가 아니면 판정에 사용 금지.

## 🗺️ 용어·테이블·파일 매핑표

### A. 테이블 매핑 (newsamc ↔ 개발계획서 v6)

| newsamc | 개발계획서 v6 | 매핑 방침 |
|---------|--------------|----------|
| `allowed_ingredients` | `ingredient_list` | **SAMC_NEW는 `ingredient_list` 사용**. newsamc의 INS/CAS/scientific_name 컬럼은 이미 개발계획서에도 존재. `allowed_status` enum은 신규 도입 검토 (§01 참고) |
| `forbidden_ingredients` | **없음** | **신규 추가 제안** — 식품유형 무관 절대 금지 게이트. 개발계획서 §4-1 Step1 "수입불가 판정 시 파이프라인 즉시 중단" 과 일치 |
| `additive_limits` | `thresholds` (일부) | 첨가물 `max_ppm`·`combined_group`·`conversion_factor` 스키마가 더 상세. `thresholds` 보강 or 분리 테이블 검토 |
| `safety_standards` | `thresholds` (일부) | `standard_type`=microbe/heavy_metal/pesticide/contaminant. `max_limit`은 TEXT(불검출/음성 등) |
| `ingredient_synonyms` | `ingredient_list.aliases[]` | 개발계획서는 배열 컬럼, newsamc는 별도 테이블. **배열 유지 권장** (join 비용 절약) |
| `food_types` | **암묵적** | 기능2(아람) 영역. 기능1은 입력으로만 받음 |
| `allergens` | `allergy_list` | **세연 담당 영역.** 기능1의 후처리에서만 참조 |

## 🎯 병찬의 최종 산출 (기능 1)

개발계획서 §4-1 기준:

**입력**
- 원재료 목록 `[{name, percentage, ins, cas, chemical_name}]`
- 제품 정보 `{product_name, alcohol_percentage, is_heated, is_fermented, ...}`
- (기능2 결과 식품유형 수신 후 재실행 케이스도 있음)

**출력** (pipeline_steps.ai_result JSONB)
```json
{
  "ingredient_verdicts": [...],
  "aggregation": {"total":..., "permitted":..., "restricted":..., "prohibited":..., "unidentified":...},
  "conditional_evaluations": [...],
  "standards_check": {"results":[...], "compound_results":[...], "has_violations":...},
  "import_possible": true,
  "verdict": "수입가능",
  "law_refs": [{"law_source":"...", "law_article":"..."}]
}
```

**판정 규칙** (개발계획서 §4-1 + newsamc 보강)
- `prohibited`(별표3/forbidden_ingredients) 1건이라도 → **수입불가, 파이프라인 중단**
- `unidentified` 1건이라도 → **담당자 확인 필요** (에스컬레이션)
- `restricted` 조건부 충족 실패 → **수입불가 또는 확인 필요**
- `standards` 위반(개별 또는 compound 합산) → **수입불가**
- 위 모두 통과 → `import_possible=true`
