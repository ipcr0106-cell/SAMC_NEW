## 4. E2E 테스트 (데모 시연 시나리오)

### 4-1. 골든 패스

1. **정상 통과**: `데모_정상_건.json` → 최종 "수입 가능" PDF 출력
2. **수입 불가 (포지티브)**: `데모_포지티브_탈락.json` → 별표3 원료 포함 → 최종 불가
3. **수입 불가 (기준치)**: `데모_기준치_초과.json` → 안식향산 0.08% → 최종 불가

### 4-2. 데모용 샘플 데이터

```
테스트용_자료/
├── 데모_정상_건/
│   ├── 원재료배합비율표.pdf
│   ├── 제조공정도.pdf
│   └── 제품_기본정보.json
├── 데모_포지티브_탈락/
│   └── ...
└── 데모_기준치_초과/
    └── ...
```

### 4-3. E2E 실행 절차 (수동)

```
1. 새 건 생성 (제품명 + 수입업체)
2. 서류 업로드 (3종)
3. 파싱 완료 확인
4. 기능2 실행 → 식품유형 확정 → confirm
5. 기능1 실행 → 결과 확인 → 판정 편집 → confirm
6. 기능3·4·5 진행
7. 최종 PDF 다운로드
```

## 5. 데이터 품질 검증 (SQL)

```sql
-- 1. ingredient_list 필수 컬럼 누락 없는지
SELECT COUNT(*) FROM ingredient_list WHERE law_source IS NULL;  -- 0이어야

-- 2. restricted 에 usage_condition 누락 없는지
SELECT COUNT(*) FROM ingredient_list
 WHERE allowed_status='restricted' AND usage_condition IS NULL;  -- 0

-- 3. thresholds is_verified 체크
SELECT is_verified, COUNT(*) FROM thresholds GROUP BY is_verified;
-- 판정 사용 행은 모두 is_verified=true

-- 4. forbidden_ingredients 최소 요구치
SELECT COUNT(*) FROM forbidden_ingredients;  -- ≥ 8

-- 5. trgm 성능 (100ms 이내)
EXPLAIN ANALYZE SELECT * FROM search_ingredients_trgm('쌀가루', 10);

-- 6. 중복 체크
SELECT name_ko, COUNT(*) FROM ingredient_list
 GROUP BY name_ko HAVING COUNT(*) > 1;  -- 0이어야

-- 7. 복합 합산 일관성
SELECT compound_group, COUNT(*) FROM thresholds
 WHERE compound_group IS NOT NULL
 GROUP BY compound_group;
-- is_compound_limit=true 1건 + 멤버 N건
```
