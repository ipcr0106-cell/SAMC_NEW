## 3. SQL 전문 (순서대로)

### 3-1. PR-01: pg_trgm 확장

```sql
-- schema.sql 상단 (다른 CREATE TABLE 이전)
CREATE EXTENSION IF NOT EXISTS pg_trgm;
```

### 3-2. PR-02: allowed_status 컬럼

```sql
-- ▼ schema.sql — ingredient_list 테이블 CREATE 바로 아래
ALTER TABLE ingredient_list
  ADD COLUMN IF NOT EXISTS allowed_status TEXT NOT NULL DEFAULT 'permitted'
    CHECK (allowed_status IN ('permitted','restricted','prohibited'));

-- 기존 데이터 마이그레이션 (is_allowed → allowed_status)
UPDATE ingredient_list
   SET allowed_status = CASE
       WHEN is_allowed = false THEN 'prohibited'
       ELSE 'permitted'
   END
 WHERE allowed_status = 'permitted'  -- DEFAULT 상태인 행만
   AND is_allowed IS NOT NULL;
```

### 3-3. PR-03: forbidden_ingredients

```sql
CREATE TABLE IF NOT EXISTS forbidden_ingredients (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name_ko     TEXT NOT NULL,
  name_en     TEXT,
  aliases     TEXT[] NOT NULL DEFAULT '{}',
  category    TEXT NOT NULL
              CHECK (category IN ('drug','endangered','unauthorized','toxin','other')),
  law_source  TEXT,
  reason      TEXT,
  is_verified BOOLEAN NOT NULL DEFAULT false,
  created_by  UUID REFERENCES auth.users(id),
  created_at  TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at  TIMESTAMPTZ NOT NULL DEFAULT now()
);

COMMENT ON TABLE forbidden_ingredients IS
  '식품유형 무관 절대 금지 원재료 마스터 — 기능1 Step 0 게이트';

CREATE INDEX idx_forbidden_ingredients_name_ko
  ON forbidden_ingredients (name_ko);

CREATE INDEX idx_forbidden_ingredients_name_ko_trgm
  ON forbidden_ingredients USING gin (name_ko gin_trgm_ops);

-- updated_at 트리거 연결 (schema.sql 말미의 공통 함수 재사용)
CREATE TRIGGER trigger_forbidden_ingredients_updated_at
  BEFORE UPDATE ON forbidden_ingredients
  FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

### 3-4. PR-04: thresholds 컬럼 확장

```sql
ALTER TABLE thresholds
  ADD COLUMN IF NOT EXISTS conversion_factor NUMERIC(8,6),
  ADD COLUMN IF NOT EXISTS colorant_category TEXT
    CHECK (colorant_category IN ('tar','non-tar','natural') OR colorant_category IS NULL),
  ADD COLUMN IF NOT EXISTS standard_type TEXT
    CHECK (standard_type IN ('additive','microbe','heavy_metal','pesticide','contaminant')
           OR standard_type IS NULL),
  ADD COLUMN IF NOT EXISTS threshold_text TEXT;

COMMENT ON COLUMN thresholds.conversion_factor IS
  '염→산 환산계수 (예: 안식향산나트륨 → 안식향산 0.847)';
COMMENT ON COLUMN thresholds.threshold_text IS
  '"불검출","음성" 등 비수치 기준. NOT NULL이면 threshold_value 무시';
COMMENT ON COLUMN thresholds.standard_type IS
  '기준 유형 분류. additive=첨가물, microbe=미생물, heavy_metal=중금속 등';
```

### 3-5. PR-05: trgm 인덱스 + RPC

```sql
-- GIN 인덱스 3종
CREATE INDEX IF NOT EXISTS idx_ingredient_list_name_ko_trgm
  ON ingredient_list USING gin (name_ko gin_trgm_ops);

CREATE INDEX IF NOT EXISTS idx_ingredient_list_name_en_trgm
  ON ingredient_list USING gin (name_en gin_trgm_ops);

CREATE INDEX IF NOT EXISTS idx_ingredient_list_scientific_trgm
  ON ingredient_list USING gin (name_scientific gin_trgm_ops);

-- 검색 RPC
CREATE OR REPLACE FUNCTION public.search_ingredients_trgm(
  q TEXT,
  k INTEGER DEFAULT 30
)
RETURNS TABLE (
  id UUID,
  name_ko TEXT,
  name_en TEXT,
  name_scientific TEXT,
  allowed_status TEXT,
  usage_condition TEXT,
  similarity REAL
)
LANGUAGE sql STABLE AS $$
  SELECT
    il.id, il.name_ko, il.name_en, il.name_scientific,
    il.allowed_status, il.usage_condition,
    GREATEST(
      COALESCE(similarity(il.name_ko, q), 0),
      COALESCE(similarity(il.name_en, q), 0),
      COALESCE(similarity(il.name_scientific, q), 0)
    )::real
  FROM ingredient_list il
  WHERE il.name_ko % q
     OR (il.name_en IS NOT NULL AND il.name_en % q)
     OR (il.name_scientific IS NOT NULL AND il.name_scientific % q)
  ORDER BY 7 DESC    -- similarity
  LIMIT GREATEST(k, 1);
$$;
```

## 4. 실행 절차 (팀컨벤션 §8 준수)

각 PR마다:

1. **채팅 공지** 템플릿:
   ```
   [스키마 변경 예고 — 기능1 병찬]
   테이블: <테이블명>
   변경 내용: <요약>
   영향: <타 기능 영향>
   PR: <URL>
   ```
2. **PR 오픈 + 전원 Reviewer 지정**
3. Reviewer 승인 수집 (최소 4/5인, 팀컨벤션 "전원 approve")
4. **머지 → 본인이 Supabase 대시보드에서 SQL 실행**
5. 실행 성공 시 채팅에 **"완료 공지"** + 검증 쿼리 결과 스크린샷

### 실행 후 검증 쿼리

```sql
ALTER TABLE thresholds
  DROP COLUMN IF EXISTS conversion_factor,
  DROP COLUMN IF EXISTS colorant_category,
  DROP COLUMN IF EXISTS standard_type,
  DROP COLUMN IF EXISTS threshold_text;
```
