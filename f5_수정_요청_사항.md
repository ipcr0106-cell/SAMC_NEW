## 3. 프론트엔드 PDF 업로드 UI — f0와 중복

현재 `LabelDraftPage.tsx`에 PDF 업로드 기능이 내장되어 있습니다. 하지만 우리 파이프라인에서는 **f0에서 이미 서류 업로드+OCR이 완료된 상태**에서 F5가 시작됩니다.

**선택지:**
- A) PDF 업로드 UI 제거 → "시안 생성 실행" 버튼만 남김 (f0 OCR 결과 자동 사용)
- B) PDF 업로드 유지 → 사용자가 추가 서류를 올릴 수 있도록 (현재 상태)

F5는 파이프라인 마지막 단계이므로, 이전 단계 결과가 부족할 때 추가 서류를 올릴 수 있는 B도 나쁘지 않습니다. 세연님이 판단해주세요.

## 5. f0 ~ F4 → F5 데이터 파이프라인 — PM이 임의로 구현한 부분

> **주의**: 이 연결은 PM이 병합 과정에서 임의로 작성한 것입니다.

### 변경 내용

`backend/routers/feature5.py`에 파이프라인 자동 연결이 추가되었습니다:

1. **food_type 자동 보충**: body에 food_type이 없으면 F2 결과에서 자동 조회
2. **파이프라인 컨텍스트 주입**: f0~F4 결과를 합쳐서 documents 목록에 추가 → AI가 이전 기능 결과를 참고

**주입되는 컨텍스트:**
```
[f0] 제품명, 원산지, OEM 여부, 원재료 목록
[F1] 수입판정 결과 (verdict)
[F2] 식품유형, 주류 여부
[F4] 라벨 검토 확정 지적사항 전체 (텍스트 + 이미지 위반)
```

### F5 출력값 (pipeline_steps step_key='6')

```
├── phase1.items[]          12개 필수항목 법령 대조
│   ├── field               "제품명" / "식품유형" / ...
│   ├── law_ref             "식품 등의 표시기준 제4조"
│   ├── status              "pass" | "fail" | "unclear"
│   └── note                판정 근거
├── phase2
│   ├── validation[]        Phase 1 교차검증
│   ├── additional_issues[] 추가 이슈
│   └── draft               ← 최종 한글시안
│       ├── product_name, food_type, ingredients
│       ├── net_weight, expiry, storage
│       ├── manufacturer, importer
│       ├── allergy, gmo, country_of_origin
└── confirmed_by            담당자 확정 시 이름
```

### 수정하고 싶다면

`backend/routers/feature5.py`에서 PM이 추가한 함수 3개:

```
def _fetch_pipeline_result(case_id, step_key) → dict
  └─ pipeline_steps 조회

def _enrich_food_type(case_id, body_food_type) → str
  └─ F2에서 food_type 자동 가져오기

def _build_context_from_pipeline(case_id) → str
  └─ ⭐ 파이프라인 컨텍스트 구성. 바꾸고 싶으면 이 함수를 수정
```

**완전히 걷어내고 싶다면:**
1. 위 3개 함수 삭제
2. `run_pipeline`에서 `_enrich_food_type` 호출 + `_build_context_from_pipeline` 호출 제거
3. 기존 세연 원본과 동일하게 동작 (documents.parsed_md만 사용)

## [공통] 법령 DB 전처리 — HWPX/PDF 파일 처리 확인

법령 원본 파일이 HWPX(한글) 또는 PDF 형식으로 제공됩니다. 자기 기능의 법령 DB 전처리 스크립트가 **두 형식 모두 처리 가능한지** 확인해주세요.

**확인 사항:**
- PDF 파싱: `pdfplumber`, `PyMuPDF(fitz)`, `pdfminer` 등 사용 가능한지
- HWPX 파싱: `python-hwp`, 외부 파서 서비스, 또는 직접 XML 파싱 가능한지
- 전처리 스크립트에서 파일 확장자에 따라 분기 처리가 되어 있는지
- 법령 업데이트 시스템(`/admin/law-update`)에서 PDF/HWPX 업로드 시 자기 기능의 전처리가 정상 트리거되는지

**현재 상태:**
- f0: PDF(PyMuPDF) + HWPX(외부 파서 서비스) 지원
- F4: PDF(PyMuPDF) + HWPX 지원 (`preprocess_laws.py`)
- F1/F2/F3/F5: 각 담당자가 확인 필요
