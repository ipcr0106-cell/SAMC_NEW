## 1. F1/F2 → F4 자동 연결 구현됨 — PM이 임의로 구현한 부분

> **주의**: 이 연결은 PM이 병합 과정에서 임의로 작성한 것입니다.
> F4 담당자가 검토 후 마음에 안 들면 자유롭게 수정하세요.

### 변경 내용

`backend/routers/feature4.py`의 `_run_analysis()` 함수에 F1/F2 자동 보충 로직이 추가되었습니다.

**기존**: `food_type`과 `ingredients`를 프론트에서 직접 전달해야 했음 (기본값: `"미분류"`, `[]`)
**변경**: 값이 비어있으면 pipeline_steps에서 F2/F1 결과를 자동 조회

```
F2 (step_key='2') → food_type 자동 보충
  - req.food_type이 "미분류"이면 → F2 결과에서 food_type 가져옴

F1 (step_key='1') → ingredients 자동 보충
  - req.ingredients가 비어있으면 → F1 결과에서 원재료 이름 목록 가져옴
```

### F4의 전체 데이터 파이프라인 (현재 상태)

```
f0 (step_key='0') ─── 연결됨 (기존)
├── product_name      → _fetch_doc_ocr()
├── ingredients       → _fetch_doc_ocr()
├── export_country    → _fetch_doc_ocr()
├── label_texts       → _fetch_label_ocr()
├── label_images      → case_label_images 테이블
├── content_volume    → ❌ 미연결 ("" 하드코딩, f0에 필드 추가 필요)
└── manufacturer      → ❌ 미연결 ("" 하드코딩, f0에 필드 추가 필요)

F1 (step_key='1') ─── 연결됨 (PM 추가)
└── ingredients[].name → req.ingredients 자동 보충

F2 (step_key='2') ─── 연결됨 (PM 추가)
└── food_type         → req.food_type 자동 보충
```

### 수정하고 싶다면

PM이 추가한 코드는 `backend/routers/feature4.py`에 있는 함수 2개입니다:

```
def _fetch_f1_result(supabase, case_id) → dict
  └─ pipeline_steps(step_key='1')에서 F1 결과 조회

def _fetch_f2_result(supabase, case_id) → dict
  └─ pipeline_steps(step_key='2')에서 F2 결과 조회
```

호출부는 `_run_analysis()` 함수 안:
```python
# F2 → food_type 자동 보충
if req.food_type == "미분류":
    f2 = _fetch_f2_result(sb, case_id)
    ...

# F1 → ingredients 자동 보충
if not req.ingredients:
    f1 = _fetch_f1_result(sb, case_id)
    ...
```

**완전히 걷어내고 싶다면:**
1. `_fetch_f1_result`, `_fetch_f2_result` 함수 삭제
2. `_run_analysis` 안의 `# F2 → food_type` / `# F1 → ingredients` 블록 삭제
3. 기존 성은 원본과 동일하게 동작 (프론트에서 직접 전달 방식)
